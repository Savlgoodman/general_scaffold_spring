# Admin Backend - Spring Boot 项目计划

基于 Spring Boot 的后台管理系统脚手架，集成完整 RBAC 权限系统，支持用户管理、角色权限、菜单管理、API 日志、操作审计、异常监控、通知公告等功能。前端通过 OpenAPI Schema 自动生成调用代码，实现无感对接。

|------|---------|------|------|

| 安全框架 | Spring Security | — | 认证与授权 |
| 参数校验 | Spring Validation | — | JSR 380 Bean Validation |
| 数据库 | PostgreSQL + HikariCP | — | 关系型数据库 + 默认连接池 |
| 数据库迁移 | Flyway | — | SQL 脚本式迁移管理 |
| 缓存/会话 | Spring Data Redis + Lettuce | — | Token 黑名单、验证码存储、缓存 |

| 图形验证码 | easy-captcha (com.github.whvcse) | — | 支持算术/GIF/中文等类型 |

## 功能模块与实现方案

### 第一阶段：基础设施

#### 1.1 项目骨架搭建
- 完善 pom.xml，引入全部依赖
- 配置 application.yml 多环境（dev/test/prod）
- 配置 Flyway 初始化迁移脚本
- 统一响应体 `R<T>`、分页封装 `PageResult<T>`
- 全局异常处理器 `GlobalExceptionHandler`
- BaseEntity 基类（id, createTime, updateTime, isDeleted），createTime/updateTime 自动填充
- MyBatis-Plus 配置（分页插件、自动填充、逻辑删除）

#### 1.2 数据库设计与迁移
- Flyway 管理所有表结构变更
- 迁移脚本存放于 `src/main/resources/db/migration/`
- 命名规范：`V1__init_schema.sql`, `V2__add_notices.sql`

核心表（全部以 `admin_` 为前缀）：
- `admin_user` — 管理员用户
- `admin_role` — 角色
- `admin_permission` — 权限
- `admin_menu` — 菜单（树形，parent_id 自关联）
- `admin_user_role` — 用户-角色关联
- `admin_role_permission` — 角色-权限关联
- `admin_role_menu` — 角色-菜单关联
- `admin_user_permission_override` — 用户级权限覆盖（grant/deny）
- `admin_api_log` — API 请求日志
- `admin_login_log` — 登录日志
- `admin_operation_log` — 操作审计日志
- `admin_error_log` — 系统异常日志
- `admin_notice` — 通知公告

所有表包含软删除字段 `is_deleted`，通过 MyBatis-Plus `@TableLogic` 全局生效。

### 第二阶段：认证与权限

#### 2.1 JWT 认证
- `java-jwt` 签发 Access Token + Refresh Token
- Access Token 短期有效（如 30 分钟），Refresh Token 长期（如 7 天）
- Token 黑名单存 Redis（登出时写入，过期自动清除）
- `JwtAuthenticationFilter` 拦截请求，校验 Token 有效性
- Spring Security 配置：放行登录/注册/验证码接口，其余需认证

#### 2.2 图形验证码
- 使用 `easy-captcha` 生成验证码图片
- 流程：生成 captchaKey(UUID) → 验证码值存 Redis（TTL 5 分钟）→ 图片 Base64 返回前端
- 登录时校验 captchaKey + captchaCode，校验后立即删除 Redis key

#### 2.3 登录安全
- 密码使用 `BCryptPasswordEncoder` 加密存储
- 登录失败计数：Redis key `login:fail:{username}`，累计 N 次后锁定账户（TTL 自动解锁）
- 每次登录（成功/失败）写入 `login_log` 表

#### 2.4 RBAC 权限系统
- 标准三层：用户 → 角色 → 权限
- 菜单可见性：用户 → 角色 → 菜单
- 用户级权限覆盖：`user_permission_override` 表，effect 字段（GRANT/DENY）
- 权限匹配支持通配符：使用 Spring 的 `AntPathMatcher`
- 超级管理员角色：`role.code = 'SUPER_ADMIN'`，绕过所有权限校验
- 动态权限同步：启动时扫描所有 `@Operation` 注解，自动同步到 permission 表

### 第三阶段：日志体系

#### 3.1 API 请求日志
- AOP 切面 `ApiLogAspect`，拦截所有 Controller 方法
- 记录：请求路径、方法、参数、响应码、耗时、用户ID、IP、User-Agent
- 异步写入 `api_log` 表（避免影响接口性能）

#### 3.2 登录日志
- 在登录 Service 中直接记录
- 记录：用户名、登录时间、IP、User-Agent、登录结果（成功/失败原因）

#### 3.3 操作审计日志
- 自定义 `@OperationLog(module = "用户管理", type = OperationType.UPDATE)` 注解
- AOP 切面 `OperationLogAspect` 处理
- 记录：操作人、操作时间、模块、操作类型（增/删/改）、操作前数据、操作后数据

#### 3.4 系统异常日志
- `GlobalExceptionHandler` 中捕获未处理异常
- 记录：异常类型、异常消息、堆栈信息、请求路径、请求参数、用户ID、IP
- WARNING/ERROR/CRITICAL 级别异常自动入库

### 第四阶段：业务功能

#### 4.1 用户管理
- CRUD：列表（分页+搜索）、详情、新增、编辑、删除（软删除）
- 分配角色：批量设置用户角色
- 权限覆盖：为指定用户单独 grant/deny 权限
- 重置密码

#### 4.2 角色管理
- CRUD：列表、详情、新增、编辑、删除
- 分配权限：批量设置角色权限
- 分配菜单：批量设置角色可见菜单

#### 4.3 权限管理
- CRUD：列表、详情、新增、编辑、删除
- 权限同步：从 OpenAPI 注解自动同步

#### 4.4 菜单管理
- CRUD：列表（树形结构）、新增、编辑、删除
- 树形查询：返回嵌套树结构
- 字段：名称、路径、图标、排序、父级ID、类型（目录/菜单/按钮）

#### 4.5 通知公告
- CRUD：列表（分页）、详情、新增、编辑、删除
- 状态流转：草稿 → 已发布 → 已撤回
- 置顶与排序
- 发布时间管理

#### 4.6 系统监控
- OSHI 采集：CPU 使用率、内存使用、磁盘空间
- Redis 连接状态与信息
- 数据库连接池状态
- JVM 运行信息

### 第五阶段：文档与规范

#### 5.1 OpenAPI 文档规范（关键）
前端依赖 OpenAPI 文档生成接口调用代码，必须保证注解完整一致：

- 每个 Controller 类：`@Tag(name = "模块名", description = "模块描述")`
- 每个接口方法：`@Operation(summary = "简要说明", description = "详细描述")`
- 每个请求/响应 DTO 字段：`@Schema(description = "字段说明", example = "示例值")`
- 错误响应：`@ApiResponse(responseCode = "4xx", description = "错误说明")`
- 枚举类型：`@Schema(enumAsRef = true)` 确保前端正确生成类型
- 统一响应体 `R<T>` 需正确暴露泛型类型

#### 5.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

状态码枚举 `ResultCode`：
- 200 成功
- 400 参数错误
- 401 未认证
- 403 无权限
- 404 资源不存在
- 423 账户锁定
- 500 服务器内部错误

关键配置项：

- `logging` — 日志级别与文件轮转
